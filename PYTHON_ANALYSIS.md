# Анализ Python скрипта app.py

## 🎯 **Основная задача скрипта:**
Автоматизировать поиск доступных слотов для практического экзамена по вождению на сайте DVSA и автоматически их бронировать.

## 🔑 **Ключевые локаторы и элементы:**

### **Страница логина:**
- `//input[@id="driving-licence-number"]` - поле номера водительских прав
- `//input[@id="application-reference-number"]` - поле референс номера 
- `//input[@id="booking-login"]` - кнопка входа
- `//section[@id="confirm-booking-details"]` - секция подтверждения после входа
- `//section[contains(@class,"error-summary")]` - блок ошибок авторизации

### **Поиск тестовых центров:**
- `//a[@id="test-centre-change"]` - кнопка изменения тест-центра
- `//input[@id="test-centres-input"]` - поле ввода названия центра
- `//input[@id="test-centres-submit"]` - кнопка поиска центра
- `//a[contains(@id, "centre-name")]` - ссылка на найденный центр

### **Поиск слотов:**
- `//a[@id="why-no-slots-help-link"]` - элемент "нет доступных слотов"
- `//td[contains(@class, "BookingCalendar-date--bookable")]//a` - доступные даты
- `//h1[text() = "Search limit reached"]` - лимит поиска достигнут
- `//a[@id="change-test-centre"]` - смена тест-центра
- `//a[@id="warning-ok"]` - подтверждение предупреждения

### **Бронирование слота:**
- `//a[@data-date="{slot_date}"]` - клик по конкретной дате
- `//input[@class="SlotPicker-slot"]/..` - выбор времени слота
- `//input[@id="slot-chosen-submit"]` - подтверждение выбора
- `//button[@id="slot-warning-continue"]` - продолжить несмотря на предупреждение

### **Система капчи:**
- `#main-iframe` - основной iframe с капчей
- `.h-captcha` - элементы hCaptcha
- `#interstitial-inprogress` - страница ожидания

## 🔄 **Алгоритм работы:**

1. **Инициализация** → Multilogin профиль + прокси
2. **Авторизация** → заполнение логин формы + решение капчи
3. **Циклический поиск по центрам** → перебор всех тест-центров
4. **Анализ календаря** → поиск доступных дат в нужном диапазоне
5. **Автоматическое бронирование** → если найден подходящий слот
6. **Уведомление** → отправка в Telegram о найденном слоте

## 📋 **Основные методы Python скрипта:**

### **Инициализация браузера (`start_browser()`):**
- Поиск существующих профилей Multilogin
- Создание нового профиля если не найден
- Обработка ошибок прокси `GET_PROXY_CONNECTION_IP_ERROR`
- Подключение к Selenium через порт

### **Загрузка главной страницы (`load_main()`):**
- Переход на `https://driverpracticaltest.dvsa.gov.uk/login`
- Ожидание элементов логина или подтверждения бронирования
- Автоматическое определение состояния авторизации

### **Авторизация (`auth()`):**
- Заполнение номера водительских прав
- Заполнение референс номера заявки
- Обработка ошибок авторизации и отмененных тестов

### **Решение капчи (`solve_captcha()`):**
- Извлечение данных hCaptcha из iframe
- Отправка на сервис решения капчи
- Передача токена обратно через `onCaptchaFinished()`

### **Ожидание с капчей (`wait_with_captcha()`):**
- Мульти-условное ожидание элементов
- Автоматическое решение капчи при появлении
- Обработка блокировок IP и страниц ожидания

### **Поиск слотов (`find_slot_script()`):**
- Циклический перебор тест-центров
- Поиск центра по названию
- Анализ календаря доступных дат
- Фильтрация дат по заданному диапазону
- Попытка бронирования при нахождении

### **Бронирование (`book_slot()`):**
- Клик по найденной дате
- Выбор первого доступного времени
- Подтверждение бронирования
- Обработка предупреждений

### **Человекоподобные действия:**
- `custom_type()` - печать с задержками между символами
- `custom_click()` - клики с паузами
- `clear_input()` - очистка полей с клавиатурными сочетаниями

## 🔧 **Конфигурационные параметры:**
- `pause_search` - пауза между поисками
- `pause_type` - пауза перед печатью
- `pause_clicks` - пауза перед кликами  
- `typing_speed` - скорость печати
- `captcha_solve_attempts` - попытки решения капчи
- `change_profile_on_ban` - смена профиля при блокировке
- `browser_type` - тип браузера (mimic/stealthfox)
